# Listing Card Theme & Layout Update

## Objective

Update the listing cards to align with the supplied design reference while preserving all existing functionality, data sources, and theme standards.

---

# Posts Page Listing Cards

## Layout Changes

The supplied design reference applies **only** to the listing cards displayed on the **Posts page**.
/home/wolf/Projects/lalela/lalela_V13/c3kOa.jpg
### Requirements

* Update the visual layout to match the supplied design reference as closely as possible.
* Do **not** change any data sources.
* Do **not** change any business logic.
* Do **not** change any field mappings.
* All existing information currently displayed on the card must continue to be displayed.
* Maintain the current functionality of menus, actions, navigation, and interactions.

### Title Position

* The listing title must remain in its current position.
* The title must **not** be moved, repositioned, or redesigned.
* The new layout should be adapted around the existing title placement.

### Data Preservation

The redesign is strictly a presentation-layer update.

The following must remain unchanged:

* Listing title source
* Description source
* User information source
* Location source
* Status source
* Image source
* Metadata source
* Navigation behaviour
* Card actions and menus

---

## Theme Usage

All styling must continue to use the existing theme variables.

### Card Background

* Use the theme **Background** color.

### Card Accents

* Use the theme **Primary** color.

### Text

* Use the theme **Text Primary** color.

### Restrictions

* No new colors may be introduced.
* No hard-coded colors may be introduced.
* No changes may be made to the application's existing color palette.
* All colors must be sourced from existing theme variables.
