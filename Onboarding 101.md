**Onboarding v1.1**

The onboarding experience in version 1.1 is more structured and deliberate, guiding each user type through a clear, role-based journey while keeping the flow efficient and purposeful. The system now emphasizes identity first, then context, then action.

---

**1. New User Creating a Community**
This is the most guided and comprehensive route. The user begins by entering their personal details—name and core profile information—followed by verification via email or mobile number. This establishes trust and identity upfront.

Once verified, the user transitions directly into a structured five-step onboarding flow focused on community creation. This process is progressive and clearly segmented:

Initial Profile ----Autho

* Step 1: Define the community name
* Step 2: Set geographic or operational coverage
* Step 3: Category associated businesses
* Step 4: Import or add businesses into the system
* Step 5: Rules

Each step is focused, with minimal friction, ensuring the user always knows where they are and what comes next. The tone is guided and constructive, helping the user build something tangible without overwhelming them.

At completion, the user is automatically assigned as the **admin** of the community. From there, they move seamlessly into managing and growing it, with no additional setup required. The transition feels like a shift from creation to ownership.

---

**2. Invited User (No Existing Profile)**

Initial Profile ----Autho   

This route is streamlined for speed and accessibility. The user enters via a join code, which immediately links them to a specific community.

Instead of a full onboarding sequence, they are prompted to enter only essential profile details. This step is quick and lightweight, designed to remove friction while still capturing necessary identity information.

Once completed, the system automatically:

* Creates their profile
* Associates them with the community
* Assigns them a member role

The tone here is welcoming and efficient—focused on getting the user into the experience as quickly as possible. There’s no unnecessary setup; the priority is immediate participation.

---

**3. Returning User**
A returning user is anyone who already has an established profile on the platform, regardless of whether they previously created or joined a community.

This experience is all about continuity. The system recognizes the user and adapts accordingly:

* If joining a new community, they simply enter a join code and are added instantly
* If they already belong to communities, they are taken directly back into their existing environment

No onboarding repetition occurs. Profile data is reused, and actions are minimized. The tone is efficient and familiar, respecting the user’s prior investment in the platform.

The result is a seamless re-entry experience where the user can immediately engage, manage, or expand their presence without interruption.

---

Across all three routes introduces clearer structure, stronger role definition, and a smoother transition from onboarding into active use.
